## 一、项目概述
**项目名称： AuroraDocs / 极光文档**
**项目描述：** AuroraDocs 是一款面向个人、团队及小型组织的**一体化知识协作与管理平台**，整合文章管理、计划跟踪、成员协同与AI辅助功能。支持多格式文档协作、静态站点生成、团队任务管理及内置对话系统，并可通过大语言模型驱动智能化操作，旨在降低团队协作门槛，提升信息流转效率。
**项目定位：** 开源、轻量、AI驱动的团队知识管理平台，专为大学社团、大学生、个人开发者及小型团队设计，解决团队协作中“资料分散、操作繁琐、成本高昂”的痛点。
**项目愿景：** 成为轻量、易用、开源且AI原生的团队协作工具，尤其服务于非技术背景用户与资源有限的小型团队。
**核心价值：**
- 零成本启动：基础功能完全开源，无需付费即可使用；
- 极简上手：10分钟掌握核心功能，告别专业软件学习成本；
- AI赋能：自然语言操作（如“写一篇活动策划文档”），替代多工具切换；
- 完全自主：文章/数据自托管，公网发布仅需自建服务器，无厂商锁定。

## 二、目标用户与痛点深度分析
### 2.1 目标用户细分
#### 核心用户群体
1. **大学社团与学生组织**
   - 典型场景：社团招新宣传、活动策划、成员管理、经费报销
   - 核心特征：预算有限（≤100元/年）、非技术背景为主、协作频次高但时间分散

2. **大学生/个人开发者**
   - 典型场景：课程笔记整理、项目文档编写、个人知识库搭建
   - 核心特征：追求高效工具、关注数据安全、乐于尝试开源软件

3. **小型开发团队（5-20人）**
   - 典型场景：项目需求文档、API接口管理、团队任务协作
   - 核心特征：注重成本控制、需要轻量解决方案、避免过度工程化

## 三、核心功能规划

### 3.1 文档协作与管理
- 多人实时/异步协作编辑 Markdown
- 支持多种编辑模式：基础 Markdown、高级组件模式、API文档模式
- 版本历史与回溯（类Git操作记录）
- 一键分享（团队内/公开链接）
- 模板中心（项目计划、会议纪要、技术方案等预设模板）
- 文档嵌入功能（支持嵌入视频、表格、代码片段等）
- 文档评论与批注系统

### 3.2 团队与任务管理
- 成员角色与权限管理
- 任务创建、分配、进度跟踪（甘特图/看板视图）
- 团队静态站点生成（自定义HTML/CSS，支持点赞、打赏、联系表单）
- 团队活动日历（共享日程、会议提醒）
- 成员绩效统计（任务完成率、贡献度可视化）
- 团队公告系统

### 3.3 AI集成与智能辅助
- 一句话生成文章、任务、成员安排
- AI Agent服务（类Clawdbot）：支持自然语言操作全平台功能
- 支持自定义API Key或购买官方Token
- 智能整理、摘要、翻译等增强功能
- AI代码审查（自动检测代码问题、生成优化建议）
- 多语言实时翻译（支持100+语言互译）
- 智能问答机器人（基于团队知识库自动回答问题）

### 3.4 沟通与扩展功能
- 团队即时通讯（私聊/群聊/频道）
- 文件网盘服务（支持自部署或购买存储）
- 多端同步（需购买服务或自建同步服务器）
- 操作日志审计
- 屏幕共享与远程协作
- 语音/视频会议功能
- 第三方工具集成（GitHub、Notion、飞书等）

### 3.5 部署与开源
- 完全开源 AGPL-3.0 license
- 支持Docker一键部署
- 公网发布需自有服务器
- 插件系统预留（未来扩展）
- 一键升级功能（自动检测新版本并更新）
- 数据备份与恢复工具
- 性能监控面板（服务器资源使用情况可视化）

## 四、技术栈
### 4.1 已确定技术栈
1. 前端框架：
	1. 主要框架：Tauri 2 + Vue3
	2. 构建工具：Vite + Cargo
	3. 包管理：pnpm
	4. 路由：Vue Router
	5. 状态管理：Pinia
	6. 网络请求：Axios
2. 后端框架
	1. 编程语言：Rust
	2. Web服务：Salvo
3. 数据库
	1. PostgreSQL
4. 容器化
	1. Docker
5. 实时协作
	1. Yjs
### 4.2 待确定技术栈
1. **UI组件库**
   - 可选方案：Element Plus、Ant Design Vue、Naive UI
   - 选择建议：优先考虑Vue3生态成熟、文档完善的组件库，Element Plus社区活跃度高，Naive UI更现代化

2. **缓存与消息队列**
   - 缓存可选：Redis、Memcached
   - 消息队列可选：RabbitMQ、Kafka、Redis Pub/Sub
   - 选择建议：Redis兼具缓存和消息队列功能，适合轻量级场景；Kafka适合高并发消息处理

3. **状态管理**
   - 可选方案：Pinia（已在前端使用）、Redux、Zustand
   - 选择建议：前端已使用Pinia，后端可考虑使用Rust的状态管理库如Actix Web内置状态管理

4. **全文搜索**
   - 可选方案：Elasticsearch、Meilisearch、Typesense
   - 选择建议：Meilisearch轻量易用，适合中小项目；Elasticsearch功能强大，适合复杂搜索场景

5. **AI接口兼容**
   - 可选方案：OpenAI API、Anthropic Claude、Llama3（开源）
   - 选择建议：优先支持OpenAI API作为标准接口，后续扩展支持其他AI模型

6. **云服务**
   - 可选方案：阿里云、腾讯云、AWS、自部署
   - 选择建议：初期可考虑自部署降低成本，后期根据用户需求提供云部署服务

7. **网盘技术**
   - 可选方案：MinIO、Nextcloud、自建文件系统
   - 选择建议：MinIO轻量高效，适合对象存储；Nextcloud功能全面，适合团队协作场景

## 六、未来规划与愿景
### 6.1 发展阶段规划
- **短期（0-6个月）：** 成为大学社团首选工具（通过校园推广，如社团招新时捆绑使用）。
  - 目标：积累1000+校园用户，建立50+校园社团合作
  - 重点：完善校园场景功能（社团招新模板、活动策划工具）

- **中期（6-12个月）：** 接入开源AI模型（如Llama3），降低用户AI成本，吸引开发者。
  - 目标：GitHub星标突破5000，建立活跃开源社区
  - 重点：开源AI模型集成、插件系统完善

- **长期（2年+）：** 构建知识库生态，允许第三方插件（如“自动生成会议纪要”插件）。
  - 目标：成为全球知名的开源团队协作平台
  - 重点：生态系统建设、企业级服务拓展

### 6.2 生态建设
- **开源社区：** 建立完善的贡献者激励机制、代码审查流程、文档体系
- **插件市场：** 开放API接口，支持第三方开发者构建插件
- **合作伙伴：** 与教育机构、开源项目、云服务商建立合作

> 愿景：让每个团队拥有自己的“知识中枢”，无需付费、无需学习，用一句话管理一切。
